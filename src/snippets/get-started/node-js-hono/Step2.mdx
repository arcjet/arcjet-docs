Add your key to a `.env.local` file in your project root.
You can also set
[`ARCJET_ENV`](https://docs.arcjet.com/concepts/environment#arcjet-env)
and other values in `.env` files.

```ini title=".env.local"
# Run Arcjet in development <https://docs.arcjet.com/concepts/environment#arcjet-env>.
ARCJET_ENV=development
# Arcjet key for your site (from <https://app.arcjet.com>).
# More info: <https://docs.arcjet.com/concepts/environment#arcjet-key>.
ARCJET_KEY=ajkey_yourkey
```

Next you need to update the dev command in your `package.json` to use the
`.env.local` file.

```json
{
  "scripts": {
    "dev": "tsx watch --env-file .env.local src/index.ts"
  }
  ...
}
```

<Aside type="note" title="env files">
  Node 20+ supports loading environment variables from a local file with
  `--env-file`. If you're using an older version of Node, you can use a package
  like [`dotenv`](https://www.npmjs.com/package/dotenv) to load the environment
  variables from a `.env.local` file.
</Aside>
