Arcjet allows you validate & verify an email address. This is useful for
preventing users from signing up with fake email addresses and can significantly
reduce the amount of spam or fraudulent accounts.

<WhatIsArcjet />

## Quick start

This guide will show you how to add email validation and verification to a
Next.js API route.

### 1. Install Arcjet

In your project root, run the following command to install the SDK:

{/* prettier-ignore */}
<Tabs>
<TabItem label="npm">
```sh
npm i @arcjet/next
```
</TabItem>
<TabItem label="pnpm">
```sh
pnpm add @arcjet/next
```
</TabItem>
<TabItem label="yarn">
```sh
yarn add @arcjet/next
```
</TabItem>
</Tabs>

### 2. Set your key

[Create a free Arcjet account](https://app.arcjet.com) then follow the
instructions to add a site and get a key. Add it to a `.env.local` file in your
Next.js project root:

```ini title=".env.local"
ARCJET_KEY=ajkey_yourkey
```

### 3. Validate an email address

The example below shows how to use Arcjet to check an email address. If the
email address is invalid or if no MX records are configured, Arcjet will return
a deny decision.

<TabItem label="TS (App)">

<TabItem label="JS (Pages Router)">
Create a new API route at `/pages/api/arcjet.js`:

<Code
  code={Step3PagesJS}

  mark={["ARCJET_KEY"]}
/>
</TabItem>
<TabItem label="Server Actions">
See our [example in the SDK reference](/reference/nextjs#server-actions).
</TabItem>
</Tabs>

### 4. Start app

Start your app and load `http://localhost:3000/api/arcjet`. The email address is
from a disposable email service, so the decision is to deny. This can be adapted
to API routes in your application which accept user submissions e.g. a signup
form.
