## Quick start

This guide will show you how to protect a SvelteKit signup form.

### 1. Install SDK

In your project root, run the following command to install the SDK:

{/* prettier-ignore */}
<Tabs>
<TabItem label="npm">
```sh
npm i @arcjet/sveltekit
```
</TabItem>
<TabItem label="pnpm">
```sh
pnpm add @arcjet/sveltekit
```
</TabItem>
<TabItem label="yarn">
```sh
yarn add @arcjet/sveltekit
```
</TabItem>
</Tabs>

### 2. Set your key

[Create a free Arcjet account](https://app.arcjet.com) then follow the
instructions to add a site and get a key. Add it to a `.env.local` file in your
SvelteKit project root:

```ini title=".env.local"
ARCJET_KEY=ajkey_yourkey
```

### 3. Protect a form

Arcjet signup form protection is a combination of the [rate
limiting](/rate-limiting/quick-start/sveltekit), [bot
protection](/bot-protection/quick-start?f=sveltekit), and [email
validation](/email-validation/quick-start/sveltekit) primitives. These are
configured using our recommended rules.

The example below is a simple email form that submits to an API route. You could
adapt this as part of a signup form.

Create the Svelte page containing the form:

Next, create the server-side code to handle the page submission:

### 4. Start app

Start your app and load `http://localhost:5173/form`. Submit the form with a
variety of email addresses and you can see how the check behaves. The requests
will also show up in the [Arcjet dashboard](https://app.arcjet.com).
