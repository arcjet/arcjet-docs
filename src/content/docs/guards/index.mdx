Guards apply Arcjet security rules **inside AI agent tool calls** and anywhere else you process untrusted input. Pass inputs directly, get a decision back. No `Request` object needed.

They are for the places HTTP middleware can't reach: inside tool handlers, between agent steps, in queue consumers, and across agentic pipelines that fan out across many services.

<Aside type="tip">
  Looking to protect HTTP route handlers and API endpoints? See the
  <Link.Page href="/get-started">quick start guide</Link.Page> for
  request-based protection with `protect()`.
</Aside>

## Get started

Install the `add-guard-protection` skill into your AI coding agent. It will detect your language, install the package, configure rules, and wire up `guard()` calls inline in your tool handlers.

Supported languages: **JavaScript / TypeScript** (`@arcjet/guard` >= 1.4.0) and **Python** (`arcjet` >= 0.7.0).

```bash
npx skills add arcjet/skills --skill add-guard-protection
```

Then describe what you want to protect — for example: *"rate limit my tool calls per user and block prompt injection"* or *"secure my MCP server"* or *"protect my queue worker"*.

## Choosing rules

| Use case | Recommended rules |
|---|---|
| AI agent tool calls | `tokenBucket` + `detectPromptInjection` |
| MCP tool handlers | `slidingWindow` or `tokenBucket` + `detectPromptInjection` |
| Scanning tool results for injection | `detectPromptInjection` (scan the returned content) |

<Aside type="caution">
  **Do not** create the Arcjet client per call — it holds a persistent
  connection. Create it once at module scope.

  **Do not** configure rules inside a function — rule configs carry stable IDs
  for server-side aggregation. Creating them per call breaks dashboard tracking
  and rate limit state.

  **Do not** forget the `key` parameter on rate limit rules — without a key,
  Guard can't track per-user limits.
</Aside>

## Use cases

### Secure AI agents

Multi-step agent workflows fan out across tool calls and sub-agents. Add guards between steps to catch injected instructions before they propagate, enforce per-user budgets across the pipeline, and prevent a single user from exhausting shared resources.

### Secure tool calls

Tools that fetch external content (web pages, APIs, documents) can return content containing injected instructions that re-enter the model context. Guard tool inputs and outputs to detect jailbreaks, instruction overrides, and role-play escapes before they reach the model.

### Secure MCP server tools

MCP servers expose tools that receive untrusted input from AI agents. Guard each tool handler to rate limit per user and detect prompt injection in tool arguments.
